# 管家系统 v3.0

> 一个替你管理所有工作状态的全职秘书。你决策，它执行。

## 安装

1. 把这个文件夹放到你喜欢的任意位置
2. 打开你的 AI 编程助手（Codex、Claude Code、Cursor 等均可），把工作目录指向这个文件夹
3. 在对话里发送 **"初始化系统"**，管家会自动完成首次入职引导

AI 会自动加载 AGENTS.md，成为你的管家。

## 这是什么

一个由三部分组成的个人 AI 管家系统：

- **管家** — 你的全职秘书，理清思路、给出建议、安排步骤、记录进度和偏好、切换任务
- **库skill** — 知识库自带的工具集（任务初始化、文档更新、归档、体检、迁移）
- **个人知识库** — 你所有笔记、记录、索引的统一存放地

> 想了解设计理念、跟现有方案的区别、技术可行性分析，读 [`产品说明书.md`](产品说明书.md)。

## 作者自述

我是做嵌入式开发的，不是搞 AI 的。

平时用 AI 辅助写代码，时间长了发现一个问题：AI 能帮我干活，但没有人帮我管"我自己"。手上同时跑着几个任务，每个做到哪了、上次做了什么决策、偏好像什么做法——这些只能自己记。每次起一个新对话，都得重新交代一遍背景和习惯，花在纠正 AI 上的时间比做正事还多。

跟朋友聊过，发现不只是我这样。

所以我试着做了一套东西，叫管家系统。想法不复杂：一个替你管理所有工作状态的全职秘书。你决策，它执行。

结构也简单，三层：管家负责调度和判断，库 Skill 是可复用的工作流，个人知识库放项目文档和偏好记录。

说起来有点不好意思——这东西不是先想清楚再做的。我是需求先行，踩一个坑改一个版本。今天这个版本号是 v3.0，但每一个大版本背后都是一个具体的坎：有旧项目要导进来才加了迁移功能，发现"复习计划"不是"项目"才改了叫"任务"，被系统把问题踢回给自己才改了交互逻辑。每一步都不是先见之明，都是事后补救。

所以中间被人问"你这不就是 skill 蒸馏吗"的时候，确实被问住了。不是因为生气，是因为发现自己做了一个月的东西，居然说不出它好在哪。

后来慢慢想清楚了。拆开看，每个部件市面上都有——skill 模板有人做，AI 记忆有人做，角色扮演也有人做。但把三样串成一个闭环：用过记下偏好，下次基于偏好给建议，偏好变了跟着更新——这个市面上确实没有。工作流不会因为你改了习惯自己跟着变，但管家会。

这是它的内核。至于外面这套实现，说实话，做得很粗糙。我不是做产品的，很多东西是边学边做的。

放出来不是要宣传什么。是希望找到有类似感受的人。如果也在用 AI 辅助开发，也觉得每次起新对话等于重来一遍，或者任务一多就容易漏——这套方法可能会有用。

如果试了觉得哪里不好、哪里用不起来，或者对方向本身有不同看法，都希望能告诉我。一个人闷头做的东西，最大的问题就是发现不了自己的问题。
