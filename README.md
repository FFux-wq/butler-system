# 管家系统 v5.0

> 你有一个目标。它替你画路标，按你的方式带你走。你决策，它执行。

## 安装

1. 把这个文件夹放到你喜欢的任意位置
2. 打开你的 AI 编程助手（Codex、Claude Code、Cursor 等均可），把工作目录指向这个文件夹
3. 在对话里发送 **"初始化系统"**，管家会自动完成首次入职引导

AI 会自动加载 AGENTS.md，成为你的管家。

不同工具的入口机制不一样：

| 工具 | 自动读取的文件 | 备注 |
|------|--------------|------|
| Claude Code | `AGENTS.md` 或 `CLAUDE.md` | 放在项目根目录即可，自动加载 |
| Cursor | `.cursorrules` | 如果不支持 AGENTS.md，把内容复制到 .cursorrules |
| 其他工具 | 查看工具的"自定义指令/系统提示"功能 | 把 AGENTS.md 的内容粘贴到工具的全局或项目级 instructions 里 |

核心原则：让你用的 AI 工具在每次对话开始时，能读到 `AGENTS.md` 的内容。具体方式看工具的文档。

## 首次初始化

把工作目录指向管家系统后，发一句 **"初始化系统"**。管家会做这几件事：

1. 读 系统介绍.md，理解自己是什么、系统由哪三部分组成
2. 读 项目内核.md，理解核心哲学——引领者和秘书长在一起，给节奏不写死
3. 在 `个人知识库/.管家/` 下创建八个工作文件（进度状态、任务队列、时间线等），全部初始化好
4. 跟你说一声准备好了，可以开始工作

整个过程大概一两分钟。初始化完之后，你就可以直接开始聊了——"帮我起个项目""这个 WiFi 模块怎么调""帮我整理一下笔记"。

## 这是什么

你有一个想法——"我想做一个智能网关""我想准备面试""我想学 FreeRTOS"。模糊的，不确定从哪开始的。你把它告诉管家。

管家帮你把想法拆成路线。你往前走，它帮你记着走到哪了、踩过什么坑、你偏好像什么方式。下次你说"继续那个项目"，它直接接上，不用你重新交代。

你用得越久，它越知道你怎么想事、怎么做事。

它不是替你做事——你出力，它支撑。你判断方向，它搭框架、陪跑、记录。有点像教练——你上场，它帮你复盘每一步。有点像相父——只不过不是替你治国，是教你带兵。

所以它解决的，表面上是"手上同时跑几个任务、每个做到哪了全靠自己记"，但根子上是另一件事：**没有一个东西在长期帮你维持思考和行动的结构。** 切换项目、遗忘决策、重新交代背景——这些都是"没有结构"的症状。

为了做到这件事，它由三部分组成：

- **管家** — 引领者 + 秘书。帮你定目标、画路线、陪你走，同时管状态、记偏好、随时切换任务
- **库skill** — 工具集。封装了地图绘制、文档更新、归档、体检、迁移五个常用流程
- **个人知识库** — 数据层。你所有笔记、记录、索引的统一存放地

拆开看，每一样市面上都有人做——skill 模板、AI 记忆、角色扮演。但三样串成闭环的没有：**用过记下偏好，下次基于偏好给建议，偏好变了跟着更新**。

## 一个例子

把工作目录指向管家系统后，对话大概是这样：

> **你**：我上次那个智能锁项目，WiFi 模块写到哪了？
>
> **管家**：看了一下进度记录，AT 指令驱动层已经写完了，MQTT 对接卡在订阅主题设计上。要继续这个吗？
>
> **你**：对。另外明天有个面试，帮我也开个复习任务。
>
> **管家**：好。WiFi 继续推进。面试复习我帮你起个新任务——你之前偏好像分阶段复习，我按这个风格来排。先做哪个？
>
> **你**：先 WiFi 吧，复习晚上再说。
>
> **管家**：明白。上次你倾向用通配符订阅减少主题数量，要继续这个方向？

> 想深入了解设计理念、跟现有方案的区别、技术可行性分析，读 [`产品说明书.md`](产品说明书.md)。

## 作者自述

我是做嵌入式开发的，不是搞 AI 的。

平时用 AI 辅助写代码，时间长了发现一个问题：AI 能帮我干活，但没有人帮我管"我自己"。每次起一个新对话，都得重新交代一遍背景和习惯。跟朋友聊过，发现不只是我这样。

所以我试着做了这套东西。想法不复杂：你有一个目标，它替你画路标，按你的方式带你走。你决策，它执行。

这套东西不是先想清楚再做的——每一个需求都是从上一个坑里长出来的。纯秘书、Kensho、知识库的格式规范、引路者和秘书合到一起之后自然发生的互相喂养……每一步都是事后补救。中间也被人问住过，怀疑过这东西到底有没有用。后来逼自己去想，终于把内核讲清楚了：**引领者 + 秘书，长在一起的。**

完整的故事写在 [`项目文档/来时路.md`](项目文档/来时路.md) 里。

放出来不是要宣传什么。是希望找到有类似感受的人。如果试了觉得哪里不好，或者对方向本身有不同看法，都希望能告诉我。

LICENSE 选了 AGPLv3。说实话一开始不懂这些，后来才想明白：我希望它像 Linux 一样存在——开源、自由使用，但改了就要回流。免费放出来不是给人拿去改改闭源卖钱的。
