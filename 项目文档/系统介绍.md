# 系统介绍

> 这个系统由哪三部分组成，为什么需要这三部分。

## 三部分

### 管家

管家是你的引领者和秘书。你有一个模糊的想法，它帮你把终点定清楚、把路线画出来、按你的节奏带你走；你在执行，它记进度、记偏好、记踩过的坑；你想换任务，它知道每个做到哪了，无缝接上。

引领者和秘书不是两个角色——是长在一起的。引领你的时候顺手管状态，管你状态的时候随时准备指下一步。

为什么需要：因为你需要有人帮你找路（画路标、拆步骤、标难点），也需要有人帮你管状态（记进度、记偏好、无缝切换）。这两件事是同一个人做的，互相喂养。

### 个人知识库

个人知识库是你所有数据和代码的统一存放地。笔记、项目文档、源代码、学习记录都在这里。它是被动的——没有智能，不会主动做什么。

知识库自带库skill（一套维护工具），用于地图绘制、文档更新、项目归档、知识库体检和知识库迁移。管家在用户需要时调用这些工具。

为什么需要：因为你的数据和代码需要有一个统一的地方存放，而不是散落在各个文件夹里。代码和文档在一起，省去了维护外部路径索引的麻烦。

### 库skill

库skill 是个人知识库自带的工具集。五个工具：地图绘制、项目文档更新、项目归档、知识库体检、知识库迁移。覆盖从画地图、推进、归档到维护知识库的全流程。

为什么需要：因为知识库如果没人维护，用久了就会乱。库skill 就是那个维护秩序的角色，管家在用户需要时调用它。

## 三者关系

```
用户
  │
  ▼
管家（跟你一起工作）
  │ 调用
  ▼
库skill（知识库自带的工具集）
  │ 管理
  ▼
个人知识库（存放数据 + 代码）
```
