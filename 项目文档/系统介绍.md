# 系统介绍

> 这个系统由哪三部分组成，为什么需要这三部分。

## 三部分

### 管家

管家是你的全职秘书。你有了想法，它帮你理清思路、给建议、等你拍板；你确定方向后，它安排步骤、一步步引导你执行；你在干活，它在旁边记进度、记偏好、记你踩过的坑；你想换任务，它知道你另一个做到哪了，无缝接上。

为什么需要：因为你需要有人帮你管你的工作状态——不是替你做，是让你不用自己记进度、记决策、记下一步该干什么。

### 个人知识库

个人知识库是你所有数据和代码的统一存放地。笔记、项目文档、源代码、学习记录都在这里。它是被动的——没有智能，不会主动做什么。

知识库自带库skill（一套维护工具），用于任务初始化、文档更新、项目归档、知识库体检和知识库迁移。管家在用户需要时调用这些工具。

为什么需要：因为你的数据和代码需要有一个统一的地方存放，而不是散落在各个文件夹里。代码和文档在一起，省去了维护外部路径索引的麻烦。

### 库skill

库skill 是个人知识库自带的工具集。五个工具：任务初始化、项目文档更新、项目归档、知识库体检、知识库迁移。覆盖从起任务、推进、归档到维护知识库的全流程。

为什么需要：因为知识库如果没人维护，用久了就会乱。库skill 就是那个维护秩序的角色，管家在用户需要时调用它。

## 三者关系

```
用户
  │
  ▼
管家（跟你一起工作）
  │ 调用
  ▼
库skill（知识库自带的工具集）
  │ 管理
  ▼
个人知识库（存放数据 + 代码）
```
