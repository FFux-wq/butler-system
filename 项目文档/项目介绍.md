# 管家系统

## 项目介绍

### 为什么要做这个项目

你用 AI 写代码。AI 帮你干活，但没有人帮你管"你"——你的任务做到哪了、下一步该干什么、上次做的决策是什么、踩过什么坑。手上的事一多，光靠自己脑子记，迟早会漏。

你需要一个秘书。一个对所有项目和任务心里有数的人。你把想法告诉它，它帮你理清思路、给出建议、等你拍板；你确定后，它一步步引导你执行、在旁边记录一切。

### 这个项目能帮你做什么

**它把两件事合在一起：**

**个人知识库** — 你所有笔记、记录、源代码和项目文档的统一存放地。代码和文档在一起，不用额外维护外部路径索引。由库skill 负责维护它的秩序。

**管家** — 你的全职秘书。理清思路、给出建议、安排步骤、引导执行、记录进度和偏好、切换任务。每次打开对话，它知道你在做什么、做到哪了，直接进入状态。

### 两者怎么配合

```
你起了一个新任务
  → 跟管家讨论计划
  → 管家记下来，引导你执行
  → 你干活时，管家从知识库调资料给你
  → 干完了，管家更新进度
  → 需要整理时，管家调用库skill 维护知识库
  → 下次打开，管家直接接上
```

知识库是管家的"资料库"和"代码库"，管家通过固定的路径指针访问它。两者独立但协同工作。

### 这个项目不做的事

- 不需要服务器、数据库、部署——就是一堆 Markdown 文件 + AI 对话
- 不追求完美才开始——边用边改
