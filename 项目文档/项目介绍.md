# 管家系统

## 项目介绍

### 为什么要做这个项目

你用 AI 写代码。AI 帮你干活，但没有人帮你管"你"。

不是帮你记进度、记决策那种"管"——而是帮你维持一个稳定的行动结构：把模糊想法变成可走的路，按你的方式带你推进，让你在多个项目之间来去自如。没人做这件事，你只有你自己。

你需要的不只是一个秘书。你需要一个人——帮你把模糊想法定成清晰目标，画好路线按你的节奏带你走，同时管着你的所有工作状态。引领者和秘书，长在一起的。

### 这个项目能帮你做什么

**它把三样东西串在一起：**

**管家** — 你的引领者 + 秘书。定终点（把模糊想法澄清成具体目标），画路线（拆步骤、标难点、绕坑），陪你走（每步都在，推进就记、卡住就分析），同时管着进度、偏好、任务切换。每次打开对话，它知道你在做什么、做到哪了，直接进入状态。

**个人知识库** — 你所有笔记、记录、源代码和项目文档的统一存放地。代码和文档在一起，不用额外维护外部路径索引。由库skill 负责维护它的秩序。

**库skill** — 知识库自带的工具集。五个工具覆盖任务全生命周期。

### 三者怎么配合

```
你起了一个新任务
  → 跟管家讨论计划（定终点 → 画路线）
  → 管家记下来，陪你一步一步走
  → 你干活时，管家从知识库调资料给你
  → 推进了，管家更新进度和偏好
  → 需要整理时，管家调用库skill 维护知识库
  → 下次打开，管家直接接上
```

知识库是管家的"资料库"和"代码库"，管家通过固定的路径指针访问它。两者独立但协同工作。

### 这个项目不做的事

- 不需要服务器、数据库、部署——就是一堆 Markdown 文件 + AI 对话
- 不追求完美才开始——边用边改

### 谁适合用

这个系统不是给所有人做的。

如果你只是偶尔用 AI 聊两句、一次只做一个项目、不需要频繁切换——你大概不需要它。

它适合的是这样一种人：手上同时跑着好几件事，有自己的一套思考方式，脑子里偶尔冒出一些"能不能把这个做出来"的念头，并且真的会去动手。这种人不需要被管理，也不需要被教怎么做事。他们缺的是一个能跟上他们节奏的东西——脑子里已经跳了三步，管家要能接住，不是从头问起。

人群不大，但需求强度很高。一旦开始同时推进多个项目、频繁切换、想法一个接一个——这东西就从"能用"变成"离不开"。
