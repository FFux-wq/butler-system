# 工作流

> 整个系统的核心工作流。适用于任何任务——做项目、复习、预研、个人计划、突发想法。

## 工作流总图

```
讨论计划 → 安排步骤 → 执行 → 记录 → 切换任务 → 恢复
```

## 详细流程

### 第一阶段：起任务

用户告诉管家想做一件事。

管家先确定任务类型（工程项目、学习复习、文档整理、方案预研、个人计划），然后跟用户讨论：目标是什么、大概分几个阶段、预计时间。

讨论结束后，管家记下任务计划和当前状态。

### 第二阶段：执行

管家告诉用户第一步该做什么。用户执行。

执行过程中：
- 用户需要资料 → 管家从知识库调
- 用户卡住了 → 管家讨论、提示、纠正
- 用户完成了 → 管家记录进度

### 第三阶段：收尾和切换

用户说"做完了"或"想换一个"。

- 如果任务完成：管家总结、询问是否要归档
- 如果只是切换：管家记录当前进度，告诉用户还有哪些任务挂着

### 第四阶段：恢复

下次对话，管家读取工作区文件，直接告诉用户"上次做到某处了，要继续吗？"

## 库skill 的介入

在以上流程中，以下节点可能调用库skill：

| 节点 | 调用的库skill |
|------|-------------|
| 起新任务时 | 任务初始化 |
| 任务中需要更新文档 | 项目文档更新 |
| 任务做完 | 项目归档 |
| 用户说"帮我整理知识库" | 知识库体检 |
| 用户说"把我现有的项目导进来" | 知识库迁移 |

## 任务切换

用户可以随时说"换一个任务"。管家知道当前在跑哪些任务、每个进度在哪。切换时不丢进度。
