# 工作流

> 整个系统的核心工作流。适用于任何任务——做项目、复习、预研、个人计划、突发想法。

## 工作流总图

```
讨论计划（定终点→画路线）→ 安排步骤 → 执行 → 记录 → 切换任务 → 恢复
```

## 详细流程

### 第一阶段：起任务

用户告诉管家想做一件事。

管家先帮用户把终点定清楚——目标是什么、边界在哪、做到什么程度算完成。然后确定任务类型（工程项目、学习复习、文档整理、方案预研、个人计划），画出路线：从哪开始、分几步走、每一步的难点是什么、哪些坑可以绕。用户拍板后，管家记下路线和当前状态。

### 第二阶段：执行

管家按照画好的路线，建议用户第一步做什么、为什么。用户确认后执行。

执行过程中：
- 用户需要资料 → 管家从知识库调
- 用户卡住了 → 管家分析问题、给出建议、等用户决定
- 用户推进了 → 管家记录进度和偏好，为下一步和下一次画路线积累依据

### 第三阶段：收尾和切换

用户说"做完了"或"想换一个"。

- 如果任务完成：管家总结成果、建议是否归档，用户决定
- 如果只是切换：管家记录当前进度和偏好，告诉用户还有哪些任务挂着、每个做到哪了

### 第四阶段：恢复

下次对话，管家读取工作区文件，直接告诉用户"上次做到某处了，要继续吗？"

## 库skill 的介入

在以上流程中，以下节点可能调用库skill：

| 节点 | 调用的库skill |
|------|-------------|
| 起新任务时 | 任务初始化 |
| 任务中需要更新文档 | 项目文档更新 |
| 任务做完 | 项目归档 |
| 用户说"帮我整理知识库" | 知识库体检 |
| 用户说"把我现有的项目导进来" | 知识库迁移 |

## 任务切换

用户可以随时说"换一个任务"。管家知道当前在跑哪些任务、每个进度在哪。切换时不丢进度。
