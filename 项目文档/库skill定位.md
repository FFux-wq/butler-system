# 库skill 定位

> 这个项目里，库skill 是什么角色。

## 核心定义

库skill 是个人知识库自带的工具集。它提供五个工具，覆盖任务全生命周期、知识库维护和迁移导入。管家在用户需要时调用这些工具。

## 五个工具

### 1. 地图绘制

面对一个没地图的东西——不管是项目、阶段还是模块——管家切换到地图绘制模式。拼画面 → 拉链路 → 验连通 → 拆阶段 → 层层下钻 → 落文档。每一层同一套逻辑，画到"能动手做"的粒度就停。

### 2. 项目文档更新

任务进行中维护项目文档。三种模式：用户说"帮我记一下"时直接补，扫代码发现空白再提醒用户补，阶段推进时同步进度。确保代码和文档同步。

### 3. 项目归档

任务做完时用。检查文档是否完整、补充缺失内容、更新跨项目索引、归档到知识库。

### 4. 知识库体检

维护知识库的整体秩序。两个阶段：
- **体检**：检查死链、路径问题、过时内容、索引完整性
- **画像**：跨项目对照分析，更新用户画像、跨项目索引、模块芯片理解等库级文件。如果库级md文件/跨项目索引.md 不存在，画像阶段会首次创建它。

### 5. 知识库迁移

把用户现有的文件（代码、笔记、项目文件夹）导入管家系统。扫描目录 → 识别类型 → 生成初始知识库结构和项目文档 → 初始化工作区。不修改原始文件，默认复制到知识库目录下。

## 库skill 不是独立的

库skill 是知识库的一部分。不是外部安装的技能——是知识库自带的维护能力。管家通过调用这些工具来保持知识库的秩序。

## 库skill 不做什么

- 不跟用户直接对话（那是管家的事）
- 不处理日常任务和进度（那是管家的事）
- 不自动运行——用户或管家触发时才执行
- 所有操作需要用户确认后才写入
