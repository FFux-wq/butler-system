# 改进方向：前端中转界面（给管家系统换一张脸）

> 记录日期：2026-07-14
> 状态：想法阶段，待验证
> 一句话：不重造引擎，只做一层前端界面，让它去驱动底下的 Claude Code / Codex。

---

## 一、想法的由来

管家系统现在跑在 VSCode 里的 Claude Code / Codex 上。功能都在，但**用起来不够方便**——它长在别人（VSCode）的壳里，交互像写代码的工具，不像在跟一个「相父」打交道。

自然会想：**能不能把它做成一个独立软件（.exe），交互随我自己设计？**

但如果从零做一个桌面应用，最难、最烦的部分是**工具调用**——AI 怎么读文件、写文件、跑命令、一步步自己推进整个 agent 循环。这块自己实现，工程量巨大。

**关键洞察**：这块最难的活，**Claude Code / Codex 已经彻底做好了**。管家系统本来就是跑在它们上面的。所以不该重造这个轮子——应该只做一层**界面壳**，把最难的 agent 能力留给 Claude Code / Codex，自己只管「长什么样、怎么交互」。

这就是「前端中转站 / 转接口」的思路。

---

## 二、核心思路

**不是重做一个系统，是给现有系统换一张脸。**

- 管家系统 = 一套 md 规则（AGENTS.md + 库skill + 知识库），**已经**跑在 Claude Code / Codex 上。
- 要做的软件 = 给这个「已经在跑的东西」换个前端界面。
- 引擎（Claude Code / Codex）不变，知识库不变，AGENTS.md 不变——**只是把交互界面从 VSCode 换成自己的窗口。**

工作量因此天差地别：不是「造引擎」，是「贴皮」。

---

## 三、技术原理

Claude Code 除了在 VSCode 里交互式使用，还能**无头（headless）运行**——从自己的程序里调它，它照样读 AGENTS.md、照样调工具读写文件，只是把结果**以数据流形式吐回来**，前端拿这个流在自己的界面里渲染。

两种接法：

- **命令行方式**：`claude -p "用户说的话" --output-format stream-json`，流式返回 JSON，前端解析后渲染。最简单，任何语言都能调，适合先做原型。
- **官方 SDK 方式**：Claude Agent SDK（TypeScript / Python），程序化跑整个 agent，工具调用、权限、流式输出都封装好。做正式产品用这个。

Codex 同理，有对应的非交互 / exec 模式。

### 架构图

```
自己的软件（自己的「相父」聊天界面 —— 只写这一层）
      │  发消息 / 收流式结果
      ▼
Claude Code / Codex（无头模式，当引擎）  ←—— 工具调用、文件读写、agent 循环全归它
      │  读写
      ▼
管家系统 md 文件夹（AGENTS.md + 个人知识库）
```

自己只负责最上面那层「皮」。中间最难的活，引擎白送。

---

## 四、唯一要提前知道的「结」

软件底下**始终得有一个 Claude Code / Codex 在跑**，它需要 API key / 登录。

- **自己用**：完全没问题，本来就装了。今天就能开始做。
- **发给别人用**：对方也得有对应环境 + key。这是这条路唯一的代价，绕不开。
  - （除非直接接模型 API，但那样又得自己管工具调用，等于回到大坑——不划算。）

对「自己用得爽一点」这个目标，这个结不构成问题。

---

## 五、建议的推进节奏

**别一上来就做 Electron / .exe，先做最小验证。**

1. **本地网页 demo**：一个跑在 localhost 的小网页（输入框 + 聊天区），后台一个小脚本，把输入喂给 `claude -p ... --output-format stream-json`，把流式结果画回网页。目标：亲眼看到「一个自己的界面在驱动管家」。
2. **打磨交互**：验证「人用界面 + 管家干活」这个组合是否真的更顺手；调整界面（任务总览、进度、切换等）。
3. **包成桌面软件**：用 Electron / Tauri 把网页包成 .exe，那时「网页」就变成「桌面软件」，只是加一层壳。

先跑通第 1 步，再决定要不要继续投入。

## 六、待定 / 待决策

- [ ] 底层引擎选 **Claude Code** 还是 **Codex**？（初步倾向 Claude Code：headless / SDK 支持最成熟、文档最全）
- [ ] 前端技术栈：先纯网页（HTML/JS）做原型，后期 Electron 还是 Tauri？
- [ ] 界面要呈现哪些管家专属信息？（任务总览、每个任务进度、偏好、下一步建议……）
- [ ] 是否要保留「发给别人用」的可能性？这会影响要不要处理引擎/key 的分发问题。
