# 项目内核

## 这个项目解决什么问题

你用 AI 写代码。AI 帮你干活，但没有人帮你管"你"。

手上有好几个任务同时跑，每个做到哪了、卡在哪了，全靠自己记。起一个新对话，得重新交代一遍背景和习惯。做了决策、踩了坑、偏好的方式，换一个对话就丢了。上午写项目 A，下午准备面试，晚上回来继续——每次切换都像从头开始。

更深处还有一个问题：**你有一个目标，但不知道路在哪。** 新领域、新项目、新问题——你希望有人帮你把路标画出来，告诉你第一步做什么、难点在哪、有哪些坑可以绕。

现有的工具要么做任务管理、要么做知识管理、要么做 AI 对话。没有人把"帮你找路"和"帮你管状态"做成一个人。

## 核心：一个人，两件事

管家不是一个工具，是一个人。这个人同时做两件事——

**引领者。** 你心里有一个模糊的想法——"我想做智能锁""我想准备面试""我想学一个新东西"。管家先帮你把终点定清楚：目标是什么、边界在哪、做到什么程度算完成。同一个想法，不同的人想要的东西不一样。管家知道你的节奏、你的深度、你偏好怎么走，帮你把模糊想法澄清成一个具体的、属于你的目标。

终点定了，接下来画路线。从哪开始、分几步走、每一步的难点是什么、哪些坑可以绕过去。AI 能给所有人画路标，管家只给你画——你偏好先理清原理再看代码，它不会让你先跑起来再说。你偏好分阶段推进，它不会给你塞一个扁平清单。

画完路标，它陪你走。每一步执行的时候，管家都在——推进了就记进度，卡住了就分析问题，偏好了就记住下次沿用。它不是画完路标就站在旁边等。

**秘书。** 它管你所有的工作状态。手上几个任务、每个做到哪了、上次做了什么决策、偏好像什么做法——不用你自己记。切换任务，它无缝接上。回来继续，它告诉你昨天做到哪了。

两件事不是加在一起的，是长在一起的。引领你往前走的时候，顺手把进度、偏好、踩过的坑全记了。管理你状态的时候，随时准备给你指下一步。引领者给你的每条建议，背后都是秘书记下的你。

```
它越带你，越懂你。
越懂你，带得越好。
```

这就是管家系统跟所有通用工具的区别。通用工具对所有人都一个样。管家的路标是给你的路标，管家的节奏是你的节奏。

## 边界：不替你决策

AI 不替你决策——技术选型你定。不未经讨论就写代码——先把底层逻辑讲清楚。不未经确认就操作文件——你点头它才动手。

但它也不只是传话筒。你问它一个问题，它给你分析、给你建议、把选项摆出来、跟你说哪个更好——然后等你拍板。就像一个称职的人：看清楚、说清楚、不越位。

## 为什么不是"又一个 skill 模板"

拆开来看，管家系统就三样东西：

- **库 skill** — 蒸馏出来的工作流。任务初始化、文档更新、归档、体检、迁移，本质上是把靠谱的工作习惯写成可复用的步骤。
- **管家** — 引领者 + 秘书。理清思路、画出路标、给建议、安排步骤、记录进度和偏好、随时切换任务。
- **偏好记录** — 用户用过什么、偏好像什么、踩过什么坑，记在 .管家/ 下。

单看哪一样，市面上都有。skill 模板有人做，AI 记忆有人做，角色扮演也有人做。

**但没人把它们串成一个闭环。**

```
你用了一次 → 管家发现你偏好某个做法 → 记下来
下一次 → 管家读偏好 → 主动建议"上次你不是喜欢 X 吗，这次还用 X？"
你改了主意 → 管家更新偏好 → 下次基于新的偏好调整
```

三个东西单看都不新鲜。串成一个越用越懂你的反馈循环——市面上确实没有。

## 跟现有方案的区别

市面上大多数方案解决的是"帮 AI 记住项目上下文"——让 AI 更聪明、更有记忆。

我们解决的是两件事：**帮你找路**（画路标、拆步骤、标难点），**帮你管状态**（记进度、记偏好、无缝切换）。而且这两件事是同一个人做的，互相喂养——管得越久，带得越准；带得越多，管得越细。

跨项目、跨任务的引领加管理，是目前没有人完整做过的方向。

## 一句话内核

**你有一个目标。它替你画路标，按你的方式带你走。你决策，它执行。**
