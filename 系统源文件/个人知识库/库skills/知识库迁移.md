# 知识库迁移

> 库skill 之一。把用户现有的文件（代码、笔记、项目文件夹）导入管家系统，生成初始知识库。

## 这个 skill 是干什么的

用户的"知识库"可能是一个项目文件夹、一堆代码、一些散落的 MD 笔记——也可能什么都没有，只有代码仓库。

这个 skill 做的事：
1. 扫描用户指定的路径，识别里面有什么（代码项目、笔记、散落文件）
2. 参考 `库级md文件/库文件模板参考.md`，按管家标准格式生成初始的知识库结构
3. 从代码和文件中提取信息，填充到生成的 MD 文件中
4. 建立起工作区和索引

**不做的事**：不修改用户的原始文件（默认复制到新位置），不扫描代码跟知识库对照（那是体检的事）。

## 触发

- 用户说「帮我把这个项目导进来」「把我之前的代码整理成知识库」「我有一些东西想归到管家管」
- 完整启动时，检测到用户还没有知识库，建议"要不要把你现有的项目导入进来？"

## 流程

### 第一步：确认迁移范围

问用户：
- **你想导入什么？**（一个代码仓库、一个项目文件夹、一堆散落的文件？）
- **路径在哪？**
- **你想复制一份到知识库里，还是直接引用原始路径？**
- **这个项目/文件大概是什么类型的？**（嵌入式 MCU 项目、Python 脚本、Web 项目、学习笔记……）— 如果用户说不清楚，管家直接扫目录结构判断

### 第二步：扫描目录结构

扫用户指定的路径，识别里面的内容：

**如果有代码**：
- 这是什么类型的项目（通过文件后缀和目录结构判断：`.c/.h` = 嵌入式、`.py` = Python、`package.json` = Node……）
- 项目分几个目录、大概结构
- 有配置文件吗（Makefile、CMakeLists、uvprojx……）
- 识别芯片型号、框架、工具链（如果能从代码中推断）

**如果有 MD 笔记**：
- 有没有现成的项目文档
- 有没有旧格式标记（如 `<!-- kensho:managed -->`）
- 是项目文档还是通用笔记

**如果只有散落的文件**：
- 列出文件清单给用户，问"这些怎么归类？"

### 第三步：列出迁移计划

把扫描结果整理给用户看，问确认：

```
我扫描了你的文件，发现这是一个：
- 类型：嵌入式 MCU 项目（STM32F4，Keil + SPL）
- 目录：src/、inc/、docs/……
- 现有 MD 文件：× 个（有项目介绍、学习记录）
- 旧格式标记：有/无

我打算：
1. 把整个项目文件夹复制到知识库的 `系统源文件/个人知识库/` 下
2. 根据 `库级md文件/库文件模板参考.md` 中的模板，判断需要生成哪些项目文档
3. 从代码中提取技术栈、模块信息，生成项目介绍.md 和其他文档骨架
4. 现有 MD 笔记加上 butler 标记后放到对应位置
5. 初始化工作区和索引

可以开始吗？有要调整的吗？
```

用户确认后进入下一步。用户说不对就改，用户满意就执行。

### 第四步：执行迁移

**复制项目文件**：
- 把整个项目文件夹复制到 `系统源文件/个人知识库/` 下
- 保留原始文件结构，不修改原路径的文件
- 排除编译中间文件（build/、.o、.hex 等）和大型二进制文件

**按模板生成项目文档**：
- 读 `库级md文件/库文件模板参考.md`，逐条对照哪些模板适合这个项目
- 根据项目类型和复杂程度，判断用单层结构还是双层结构
- 从代码扫描结果提取信息，填充到生成的 MD 文档中：
  - 项目介绍.md（项目名、类型、技术栈、模块划分、目录结构）
  - 开发计划.md（骨架，内容留空后续填充）
  - 学习记录.md（空模板）
  - 踩坑记录.md（空模板）
  - 技术决策.md（空模板）
- 如果用户确认是分阶段项目，同时创建阶段子目录骨架

**处理现有笔记**：
- 如果用户有现成的 MD 笔记，复制到对应位置
- 内容中有 `<!-- kensho:managed -->` 标记的，转为 `<!-- butler:managed -->`
- 没有标记但内容明显是 AI 写的，加上 butler 标记
- 用户手写的内容，不加标记

**工作区初始化**：
- 创建 `.管家/` 目录
- 写入 `管家身份.md`
- 写入初始的进度状态和任务队列

### 第五步：报告结果

迁移完成后，告诉用户：

```
导入完成。在知识库中创建了 XXX 项目，包含：
- 项目介绍.md（已从代码提取技术栈和模块信息）
- 现有笔记 × N（已加入 butler 标记）
- 新生成的 MD 文件 × N

你现在可以：
1. 让管家读一下项目介绍，开始推进
2. 跑一次体检，进一步梳理索引和补充遗漏
3. 直接在管家对话里开始工作
```

---

## 注意事项

- 不要在用户没确认前执行任何写入操作
- 原始路径的文件不要动——默认复制到知识库目录下
- 从代码提取的信息标注"从代码推断，请确认准确性"
- 用户说不对的直接删或改，不保留
