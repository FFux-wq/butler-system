# 知识库体检

> 库skill 之一。维护知识库的整体秩序。两个阶段：体检（查死链/路径/过时/索引）和画像（跨项目对照、更新画像）。

## 写入规则（所有阶段通用）

所有 AI 写入的库级文件内容，必须用 HTML 注释标记管理区间：

```
<!-- butler:managed id=<唯一id> -->
（AI 管理的区块内容）
<!-- butler:managed-end id=<唯一id> -->
```

- 标记外的内容（用户手写部分），AI 不修改
- 用户确认准确的条目标记 `[已验证]`，下次运行时优先采信
- 用户说不对的条目直接删除，不保留

## 触发

- 用户说「帮我整理一下知识库」「知识库该体检了」「跑一次体检」时 → 执行两个阶段
- 用户说「帮我更新画像」「跑一次画像」时 → 只执行画像阶段

## 阶段一：体检

### 第一步：检查死链

遍历知识库中的索引文件，检查每一个链接指向的文件是否存在。发现死链列出给用户看。

### 第二步：检查路径问题

检查外部资源索引中的路径是否有效。发现路径不对的标记出来。

### 第三步：检查过时内容

检查库级文件中最近更新时间超过 90 天的条目。标记为「可能过时」展示给用户。

### 第四步：检查索引完整性

检查跨项目索引是否覆盖了所有已归档项目。发现遗漏列出给用户。

## 阶段二：画像（跨项目分析）

画像阶段由管家按以下流程执行。用户单独要求「跑画像」时也只走这个阶段。

### 第一步：汇总

读已有库级文件——库级md文件/项目总结.md、库级md文件/跨项目索引.md、库级md文件/用户画像.md、库级md文件/模块芯片理解.md——掌握全局。按需深入项目级文件（学习记录、踩坑记录、技术决策等）。

### 第二步：对照

五维度跨项目对比：

- A. 同类归并：技术点、工具在哪些项目反复出现
- B. 时间线排列：同一技术点在不同项目的进展
- C. 交叉验证：学习记录和后续行为是否一致
- D. 复现预警：同一个坑跨项目出现
- E. 模块芯片对照：同一芯片在不同项目的用法差异

每条对照结论必须引用具体文件。

### 第三步：报告

输出对照报告。用问句呈现，不替用户下结论。

**！！！用户审查节点！！！**

报告输出后停下来等用户确认。用户说不对的条目直接删，记入反馈日志。用户确认准确的条目标记 [已验证]。

## 用户确认后执行

### 体检修复（逐条确认）

用户逐条决定：修、忽略、稍后处理。确认后执行。

### 画像写入（确认后执行）

根据用户在审查节点中的决定执行：

- **用户说要创建的缺失文件** → 创建空模板后填充内容
- **用户说要更新的现有文件** → 更新对应内容
- **用户说跳过的文件** → 本次不处理

具体的更新目标：
- 库级md文件/用户画像.md（五部分：方法论、偏好、风格、知识地图、成长线）
- 库级md文件/模块芯片理解.md
- 库级md文件/跨项目索引.md（如果没有变化则跳过）
- 库级md文件/知识库全局地图.md 的知识点交叉索引部分（与目录树部分合并在同一文件）
- 库级md文件/项目总结.md

所有写入内容遵循写入规则中的标记约定。

## 报告格式

```
知识库体检报告
日期：YYYY-MM-DD

[体检发现]
1. 死链 × N
2. 路径失效 × N
3. 可能过时的条目 × N
4. 索引遗漏 × N

[画像状态]
上次肖像更新时间：YYYY-MM-DD
对照建议条目：N 条（待确认）

[建议处理]
每条问题给出处理建议
```
