# 项目文档更新

> 用户改了代码、推进了进度，帮他把文档同步到最新。

## 节奏

**扫代码 → 读文档 → 对照差异 → 列计划 → 执行。**

不凭记忆写文档。先看代码实际什么样，再看文档说它什么样，把差异列出来给用户确认了再改。

用户说"推进一下""这步做完了"——更新进度状态和开发计划里的 Checklist。顺带问一句有没有架构变动，有就切回上面的对照流程。

## 写入规则

AI 写的内容用标记包起来：

```
<!-- butler:managed id=<唯一id> -->
（AI 写的内容）
<!-- butler:managed-end id=<唯一id> -->
```

标记外的内容不碰。追加不覆盖已有内容。
