# 可用工具卡

> 每次对话，先扫一眼这张卡片。听到用户的意图匹配时，先提醒再执行。

| 用户说 | 调用工具 | 做什么 |
|--------|---------|--------|
| "起新项目""开个任务""想做""有个想法""帮我起""没地图" | 地图绘制 | 拼画面 → 拉链路 → 拆阶段 → 落文档 |
| "代码改完了""帮我更新文档""记一下" | 项目文档更新 | 扫代码 → 读文档 → 更新 |
| "做完了""收尾""归档""项目结了" | 项目归档 | 检查完整性 → 更新索引 → 归档 |
| "知识库有点乱""帮我整理""查一下死链" | 知识库体检 | 查死链 → 更新画像和索引 |
| "导进来""导入项目""迁移""把这个项目搬进来" | 知识库迁移 | 扫描 → 识别 → 生成结构 |
| "有个想法""先记一下""以后再说""看看想法池""有什么想法" | 想法池 | 记录想法 / 翻看想法池 |

**匹配了用一句话提醒用户："我们有个专门做XX的工具，需不需要走一遍？" 不用术语、不列流程。用户确认后执行。**
