# 管家工作台结构

> 管家每次对话结束后，把当前状态写入 .管家/ 工作区。
> 下次对话时从中读取，恢复状态。

## 工作区位置

工作区位于 系统源文件/个人知识库/.管家/ 文件夹。

## 工作区文件

### 管家身份.md

轻量启动时快速定位角色。记录摘要信息，详细版见项目文档。

### 进度状态.md

记录管家当前的项目进度和状态信息，包含 `上次更新时间` 字段用于时效性校验。格式：

```markdown
# 进度状态

## 当前状态
- **当前项目**: XXX
- **阶段**: XX
- **当前任务**: XXX
- **上次更新时间**: YYYY-MM-DD

## 项目进度
- [x] 已完成步骤
- [ ] 下一步任务
```

### 任务队列.md

所有任务的列表。格式：

```markdown
# 任务队列

## 进行中
- 项目名（描述）

## 待办
- 待办事项

## 已完成
- 已完成事项（日期）
```

### 时间线.md

按日期记录所有关键事件——项目推进、系统改造、重要决策。每次对话结束后追加本次记录。格式：

```markdown
# 时间线

## YYYY-MM-DD

### 系统改造
- 改了什么
- 改了什么

### 项目进展
- 项目名：做了什么
```

### 用户偏好.md

记录管家观察到的用户偏好。管家自己写的，不覆盖用户自己写的画像。

```markdown
- 偏好内容
- 常见卡点
- 学习/工作习惯
```

## 时效性说明

进度状态.md 中的 `上次更新时间` 字段用于入口的时效性校验（7天/30天两个阈值），管家打开新对话时据此判断是否需要先同步信息。详见入口.md。

## 注意

- .管家/ 文件夹只由管家读写，库skill 和用户不碰
- 工作区文件格式统一为 Markdown（.md），不混用 JSON
